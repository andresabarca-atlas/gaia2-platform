# GAIA Flood Impact Dashboard

Interactive dashboard for visualising GAIA flood impact results for Ecuador (pilot).
Built with Plotly Dash and deployed on Render.com.

## Running locally

```bash
cd dashboard
pip install -r requirements.txt
python app.py
```

Then open `http://127.0.0.1:8050` in your browser.

> **Data files required.** Both output files must exist at `outputs/` in the repository
> root (one level above `dashboard/`). They are tracked with **Git LFS** — run
> `git lfs pull` after cloning to download them before starting the app.

## Deploying to Render.com

1. Push the repository to GitHub (including `dashboard/render.yaml`).
2. Log in to [Render](https://render.com) and create a new **Web Service**.
3. Connect your GitHub repository — Render auto-detects `render.yaml` and configures
   the service automatically.
4. Set the `GAIA_DATA_DIR` environment variable to `/opt/render/project/src/outputs`
   (already included in `render.yaml`).
5. Trigger a deploy. The build installs dependencies from `dashboard/requirements.txt`
   and starts gunicorn.

## Output files

| File | Description |
|------|-------------|
| `outputs/adm2_flood_results.gpkg` | GeoPackage with 223 Ecuador ADM2 canton polygons. Contains total population (`pop_tot`), exposed population at each flood return period (`epop_10`–`epop_500`), and the annual average exposed population (`epop_ave`). |
| `outputs/population_points_flood.csv` | 372,065 population grid points with longitude, latitude, population count, Relative Wealth Index (`rwi`), and annual average exposed population (`epop_ave`). Only points with `epop_ave > 0` are included. |

These files are generated by the GAIA flood analysis pipeline (`run_floods_ecuador.py`).
