# GAIA Flood Impact Dashboard — Chile

Interactive dashboard for visualising GAIA flood impact results for Chile.
Built with Plotly Dash and deployed on Render.com.

## Running locally

```bash
cd dashboard
pip install -r requirements.txt
python app.py
```

Then open `http://127.0.0.1:8050` in your browser.

> **Data file required.** The output file must exist at `outputs/adm2_flood_results.gpkg`
> in the repository root (one level above `dashboard/`). Generate it by running the
> pipeline first: `python run_floods_chile.py`

## Deploying to Render.com (Chile — separate service)

This dashboard must be deployed as a **new, separate Render service** from the Ecuador
dashboard. Both can coexist because they are independent web services connected to
different branches/repos.

### Step-by-step

1. **Run the pipeline locally and commit the output.**
   ```bash
   python run_floods_chile.py
   git add outputs/adm2_flood_results.gpkg
   git commit -m "Add Chile flood results"
   git push origin chile
   ```
   > If your repo uses Git LFS for large files, make sure `outputs/*.gpkg` is tracked:
   > `git lfs track "outputs/*.gpkg"` and commit the updated `.gitattributes`.

2. **Log in to [Render.com](https://render.com).**

3. **Create a new Web Service** — click **New → Web Service**.

4. **Connect your GitHub repository** and select the **`chile` branch**.
   Render will detect `dashboard/render.yaml` automatically and pre-fill the settings.

5. **Verify the auto-detected settings** (from `render.yaml`):
   - **Name:** `gaia-dashboard-chile`
   - **Build command:** `pip install -r dashboard/requirements.txt`
   - **Start command:** `cd dashboard && gunicorn app:server --workers 1 --threads 4 --timeout 120`
   - **Environment variable:** `GAIA_DATA_DIR` = `/opt/render/project/src/outputs`

6. **Click "Create Web Service"** — Render will build and deploy.
   The URL will be something like `https://gaia-dashboard-chile.onrender.com`.

7. **Keep both dashboards running independently:**
   - Ecuador dashboard: connected to the `main` branch
   - Chile dashboard: connected to the `chile` branch

   Each branch has its own `outputs/adm2_flood_results.gpkg` — they do not interfere.

## Output file

| File | Description |
|------|-------------|
| `outputs/adm2_flood_results.gpkg` | GeoPackage with Chile ADM2 comunas. Contains total population (`pop_tot`), exposed population at each flood return period (`epop_10`–`epop_500`), and the annual average exposed population (`epop_ave`). |

Generated by `run_floods_chile.py`.
